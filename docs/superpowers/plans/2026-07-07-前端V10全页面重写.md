## TDD 分层策略(全局约束 #0,审核 v1 B5 修订)

为避免"要么所有组件严格 TDD、要么完全不测"的二元困境,本计划按组件类型分层:

| 层 | 组件类型 | 测试要求 | 理由 |
|----|---------|---------|------|
| **L1 严格 TDD** | H* 基础组件(16 个)、composable(含状态/计时逻辑)、纯函数 utils | **必须**先写失败测试 → 跑测确认失败 → 实现 → 跑测确认通过 | 行为可枚举、单测能确定性覆盖;H* 是后续所有页面复用的地基,行为错误会放大传播 |
| **L2 关键路径测试** | 跨组件状态机集成点(SessionTableV10 的 WS 进度、RecapDrawerV10 的动作按钮矩阵、SettingsView 的配置导入策略) | 写集成测试(mock store/api,断言关键交互) | 行为复杂、回归代价高 |
| **L3 视觉验证豁免** | 纯展示 section(AttentionSection/QuickActions/RecentRecapsSection/CapabilitySection 等) | **不强制单测**,靠 `npm run type-check` + dev 视觉验证 + 截图对比 V10 | 行为极简(props in、emit out、无内部状态),单测 ROI 低;类型检查兜底 props 正确性 |

**每个组件的 task 头部标注层数**(如 `[L1 严格 TDD]` / `[L3 视觉验证]`),执行者据此判断是否写测试。L3 组件允许"实现优先"——但必须包含类型检查 + dev 视觉验证两步。

## 全局约束(每个任务都遵守)

1. **类型源唯一**:`web/src/api/generated.ts`(openapi-typescript 生成)。所有新组件的 props/emit 类型、新 API wrapper 的入参出参类型,从 `generated.ts` 的 `paths` / `components['schemas']` 派生,**不再**手写 `web/src/api/types.ts`(Phase 2 起逐步迁移现有类型,Phase 6 删除 `types.ts`)。
2. **保留分层**:`api/`(契约层,无 UI 副作用)、`stores/`(实体缓存)、`composables/`(跨域复用 hook)、`features/`(业务组件)、`components/`(共享展示)、`views/`(路由壳)。**业务逻辑下沉 features,view 退化为壳**(详见 `docs/FRONTEND_ARCHITECTURE.md` §1)。
3. **保留 5 个 store + 刷新协调器**:`useAppRefreshCoordinator` 仍是 WS + 降级轮询唯一 owner。`sessionActions.ts` 状态机不变(41 单测保护)。
4. **V10 设计 token 锁定**:CSS 变量定义见 Phase 1 Task 1.2 `design-tokens.css`,**不得在组件内硬编码颜色/圆角/阴影**,只能引用 `var(--xxx)`。例外:scrollbar/skip-link 等极少纯展示元素。
5. **不动后端 API 路径/方法/payload**:除 Phase 0 显式补字段外,后端接口契约由 OpenAPI 锁定,**前端不得私改 wrapper 的 URL/method/字段名**。
6. **专家模式开关保留**:`useExpertMode` 控制高级配置/运行中任务等区块显隐。
7. **旧路径重定向保留**:router/index.ts 的 9 条 `redirect` 不变(`/live`→`/`、`/sessions/:sid`→`/recaps?sid=` 等)。
8. **每页可独立 merge**:页面间通过 store + router 解耦,任一页迁移完成后即可合并,不阻塞其他页(旧 EP view 与新 V10 view 共存期内,router 按需切换)。

```go
// 接受 *sql.Row 或 *sql.Rows(两者都有 Scan 方法,用接口约束)。
type scanner interface {
	Scan(dest ...any) error
}

func scanSessionCore(row scanner) (Session, error) {
	var s Session
	var startedAt, endedAt, uploadedAt, publishedAt, archivedAt, publishTarget, currentTaskID, lastError sql.NullString
	err := row.Scan(
		&s.ID, &s.Slug, &s.ChannelID, &s.SourceType, &s.SourceID, &s.Title,
		&startedAt, &endedAt, &s.SourceURL, &s.Status,
		&currentTaskID, &lastError, &s.LocalAvailable, &uploadedAt, &publishedAt, &archivedAt, &publishTarget,
		&s.CreatedAt, &s.UpdatedAt,
	)
	if err != nil {
		return Session{}, err
	}
	s.StartedAt = startedAt.String
	s.EndedAt = endedAt.String
	s.UploadedAt = uploadedAt.String
	s.PublishedAt = publishedAt.String
	s.ArchivedAt = archivedAt.String
	s.PublishTarget = publishTarget.String
	s.CurrentTaskID = currentTaskID.String
	s.LastError = lastError.String
	return s, nil
}

// scanSessionWithChannel 扫描 JOIN 后的 19 列(List 用,首列后多一个 channel_name)。
// 列顺序必须与 listWithChannelSQL 的 SELECT 顺序一致。
func scanSessionWithChannel(row scanner) (Session, error) {
	var s Session
	var startedAt, endedAt, uploadedAt, publishedAt, archivedAt, publishTarget, currentTaskID, lastError sql.NullString
	err := row.Scan(
		&s.ID, &s.Slug, &s.ChannelID, &s.ChannelName, &s.SourceType, &s.SourceID, &s.Title,
		&startedAt, &endedAt, &s.SourceURL, &s.Status,
		&currentTaskID, &lastError, &s.LocalAvailable, &uploadedAt, &publishedAt, &archivedAt, &publishTarget,
		&s.CreatedAt, &s.UpdatedAt,
	)
	if err != nil {
		return Session{}, err
	}
	s.StartedAt = startedAt.String
	s.EndedAt = endedAt.String
	s.UploadedAt = uploadedAt.String
	s.PublishedAt = publishedAt.String
	s.ArchivedAt = archivedAt.String
	s.PublishTarget = publishTarget.String
	s.CurrentTaskID = currentTaskID.String
	s.LastError = lastError.String
	return s, nil
```

```go
const listWithChannelSQL = `SELECT s.id, s.slug, s.channel_id, COALESCE(c.name, '') AS channel_name,
       s.source_type, s.source_id, s.title, s.started_at, s.ended_at, s.source_url, s.status,
       s.current_task_id, s.last_error, s.local_available, s.uploaded_at, s.published_at,
       s.archived_at, s.publish_target, s.created_at, s.updated_at
FROM sessions s LEFT JOIN channels c ON s.channel_id = c.id
ORDER BY s.created_at DESC, s.id DESC`
```

改 `List`(行 225)的 QueryContext 用 `listWithChannelSQL`,rows 循环里调 `scanSessionWithChannel`:
```go
func (s *Store) List(ctx context.Context) ([]Session, error) {
	rows, err := s.db.QueryContext(ctx, listWithChannelSQL)
	if err != nil {
		return nil, err
	}
	defer rows.Close()

	var sessions []Session
	for rows.Next() {
		session, err := scanSessionWithChannel(rows)
		if err != nil {
			return nil, err
		}
		sessions = append(sessions, session)
	}
	return sessions, rows.Err()
}
```

```go
	)

		return Task{}, err
	}
	t.SessionID = sessionID.String
	t.StartedAt = startedAt.String
	t.FinishedAt = finishedAt.String
	t.Error = errMsg.String
	return t, nil
}
```

```ts
// web/src/features/home/composables/useElapsedDuration.ts
import { computed, onUnmounted, ref } from 'vue'

export function useElapsedDuration(getStartedAt: () => string, tickMs = 1000) {
  const now = ref(Date.now())
  const timer = setInterval(() => { now.value = Date.now() }, tickMs)
  onUnmounted(() => clearInterval(timer))

  const text = computed(() => {
    const start = new Date(getStartedAt() || '').getTime()
    if (!start || Number.isNaN(start)) return '-'
    const sec = Math.max(0, Math.floor((now.value - start) / 1000))
    const h = String(Math.floor(sec / 3600)).padStart(2, '0')
    const m = String(Math.floor((sec % 3600) / 60)).padStart(2, '0')
    return `${h}:${m}`
  })

  return { text }
}
```
